Features

Everything you need to
collaborate with AI

loopat combines a web-based IDE, Claude-powered agent loops, and filesystem-level sandboxing into a single process you run locally.

🧶

Sandboxed Loops

Each AI session runs in its own bubblewrap mount namespace. Isolated filesystem, shared context — the best of both worlds.

💬

Collaborative Chat

WebSocket-powered real-time chat with Claude. Multiple browser tabs can share the same loop session simultaneously.

📋

Kanban Boards

Built-in Kanban with drag-and-drop, real-time sync across subscribers. Manage tasks alongside your AI agent.

💻

Web Terminal

Full PTY terminal inside the sandbox. Shared across loop subscribers — everyone sees the same bash session.

📁

File Tree & Editor

Browse and edit files inside the sandbox from the web UI. Git integration with commit and log views.

🧠

Persistent Memory

Two-tier memory system — personal (/personal/memory) and team (/context/notes/memory). Auto-recall across sessions.

Design Principles

Filesystem-first, portable, transparent

No database. Everything is a file. rsync your workspace to another machine and pick up exactly where you left off.

01

Sandbox = Directory

Sandbox permissions arenʼt hidden in config files. Theyʼre derived from the physical structure of loops/<id>/ — run ls -laR to see the full access scope.

02

Machine-Independent

rsync $LOOPAT_HOME to any machine, restart the server, and everything works — chat history, sessions, memory, sandbox views. All paths are virtual.

03

Opt-in Secrets

Sandbox canʼt see ~/.ssh, ~/.aws, or ~/.config/gh by default. Expose only what you need via explicit symlinks in personal/.

Quick Start

Up and running in seconds

One dependency, one command. Works on any Linux machine.

# 1. Install system dependency sudo apt install bubblewrap openssh-client # 2. Install Bun curl -fsSL https://bun.sh/install | bash # 3. Clone & install git clone git@github.com:simpx/loopat.git cd loopat bun install # 4. Start the server bun run dev # Open http://localhost:7787
